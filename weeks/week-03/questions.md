# Вопросы для самопроверки

1. В чем разница между Forward Proxy (обычный прокси) и Reverse Proxy (обратный прокси)?

Forward Proxy (прямой прокси) - работает на стороне клиента.

Клиент - Forward Proxy - Интернет - Сервер

Назначение:

- скрывает IP клиента

- используется для обхода блокировок

- используется в корпоративных сетях

Примеры - VPN, корпоративный proxy


Reverse Proxy (обратный прокси) - работает на стороне сервера.

Клиент - Reverse Proxy - Сервис

Назначение:

- скрывает реальные сервера

- маршрутизирует запросы

- балансирует нагрузку

Примеры - Nginx, API Gateway


2. Что такое Upstream в терминологии Nginx?

Upstream — это группа backend-серверов, к которым Nginx пересылает запросы. Используются для маршрутизации и балансировки нагрузки.

3. Зачем скрывать реальные IP-адреса сервисов от клиента?

- Безопасность

Клиент не может напрямую обратиться к сервису (например обойти защиту).

- Централизованная точка входа

Все запросы идут через один адрес.

- Гибкость

Можно менять сервисы без изменения адреса для клиента.

- Изоляция Docker сети

Контейнеры доступны только внутри docker network.

4. Какие еще задачи, кроме маршрутизации, может выполнять API Gateway (назовите хотя бы 2)?

- Аутентификация и авторизация

Проверяет токены, логин, права доступа, 

- Балансировка нагрузки

Распределяет запросы между несколькими сервисами.

- Логирование

Записывает кто когда сделал запрос, какой endpoint.

- Rate limiting

Ограничивает количество запросов.

- Кэширование

Gateway может возвращать кэшированные ответы.

5. Что произойдет, если Gateway упадет? Как можно от этого защититься?

Все сервисы станут недоступны.

Как защититься от этого: 

- Запуск нескольких Gateway

- Использование Docker restart policy

- Использование Kubernetes
