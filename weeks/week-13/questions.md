# Вопросы для самопроверки

1. Зачем нужен Helm? Какую проблему он решает?

Helm решает проблему управления сложностью и повторного использования конфигураций в Kubernetes.

Он упаковывает все эти файлы в один логический блок — Chart. Используя шаблонизатор (Go templates), он позволяет создавать универсальные манифесты, куда значения подставляются динамически из файлов values.yaml.

2. Как работает приоритет значений (Values)? Кто победит: `values.yaml` или флаг `--set`?

Приоритет значений в Helm выстраивается в четкую иерархию (от низшего к высшему):

- Значения по умолчанию в самом чарте (values.yaml внутри чарта).

- Файлы переопределения, переданные через -f или --values (например, values-dev.yaml). Если файлов несколько, последний в списке имеет больший вес.

- Флаг --set (параметры, переданные прямо в командной строке).

3. Что такое функция `include` и `_helpers.tpl`? Зачем они нужны?

_helpers.tpl - специальный файл в папке templates/, который содержит переиспользуемые фрагменты кода (named templates). Имя файла начинается с подчеркивания, чтобы Helm не пытался установить его как отдельный ресурс K8s.

include - функция, которая позволяет вставить содержимое одного шаблона в другой.

4. Почему в шаблонах Helm так важно следить за отступами и зачем нужна функция `indent` или `nindent`?

YAML — язык, критичный к отступам. Если отступы съедут, Kubernetes выдаст ошибку синтаксиса.
Когда мы используем циклы или условия, структура шаблона может нарушать отступы итогового YAML.

- indent N: Добавляет N пробелов в начале каждой строки вывода. Используется, когда нужно вставить блок кода внутрь существующей структуры с сохранением уровня вложенности.

- nindent N: (Newline + Indent). Сначала ставит перенос строки, а затем добавляет N пробелов. Это очень удобно, когда нужно вставить новый блок после какого-то ключа.

5. Как откатить неудачный релиз с помощью Helm?

Командой:

```bash
helm rollback <NAME_RELEASE> [REVISION]
```

6. Можно ли использовать Helm только для генерации YAML, без установки в кластер?

Да, можно. Команда helm template рендерит шаблоны в стандартный вывод (stdout) или в файлы, но не взаимодействует с Kubernetes API (не создает ресурсы в кластере).
