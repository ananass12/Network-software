# Диаграмма Saga

Вариант: **reviews-s18**.

**Шаги процесса:**
- **Создать заказ** в состоянии `NEW`.
- **Зарезервировать товар** на складе.
- **Списать деньги** — при успешной оплате заказ переходит в состояние `PAID`.
- **Завершить заказ** — при успешном выполнении всех операций заказ переходит в состояние `DONE`.

**Состояния и переходы:**
- Возможные состояния заказа: `NEW`, `PAID`, `DONE`, `CANCELLED`.
- Переходы:
  - `NEW` + `PAY_OK` = `PAID`
  - `NEW` + `PAY_FAIL` = `CANCELLED`
  - `NEW` + `RESERVE_FAIL` = `CANCELLED`
  - `PAID` + `COMPLETE` = `DONE`
- Терминальные состояния (не меняются дальше): `DONE`, `CANCELLED`.

**Компенсация (откат) при ошибке оплаты:**
- Если оплата не прошла (`PAY_FAIL`), необходимо отменить резерв товара и вернуть его на склад.
- Функция компенсации вызывает отмену резерва с повторными попытками (retry).
- При достижении максимального числа попыток (если задано) компенсация прекращается с ошибкой.
